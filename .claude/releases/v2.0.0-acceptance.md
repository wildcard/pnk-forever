## Random-sample acceptance

**Sampled feature:** Ren'Py emitter guard-expression translation (convertGuard).

Followed [tools/adapters/renpy/src/emit.ts](tools/adapters/renpy/src/emit.ts) `convertGuard` as documented:
- `(== $data.has_shekel false)` → `store.data_has_shekel == false` ✅
- `$data.wearing_slippers` → `store.data_wearing_slippers` ✅
- Flag name sanitization (`.` → `_`) is consistent between init block, `set_flag`, and `convertGuard` ✅

Observation: the `define config.save_directory` in [options.rpy](v2-renpy/game/options.rpy) is
a useful forward-compat hook but has no bearing on Web export (which
can't persist saves anyway). Noted — not a gap.

## Gaps filed

| ID | Priority | Title | Issue |
|---|---|---|---|
| G-01 | P2 | Ren'Py SDK download blocked in CI sandbox — web build and live-preview verifiers (RPY-04/05/06 + RPY-07 web leg) cannot run in this environment | (filed below) |

G-01 is **P2** rather than P0/P1 because:
- The pipeline (emitter, asset script, build script, Vercel config) is complete and correct.
- The blocker is environmental (SDK fetch requires network access + ~120MB download), not a defect in the shipped code.
- On a developer machine with `tools/renpy-sdk/fetch.sh` run once, `bash tools/build-renpy-web.sh` produces `dist/v2-renpy/` end-to-end.
- The next release can unblock by running fetch.sh locally and re-running the deferred verifiers.
